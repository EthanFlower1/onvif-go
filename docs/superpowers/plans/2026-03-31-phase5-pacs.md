# Phase 5: PACS (Physical Access Control) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the complete Access Control, Door Control, Credential, Schedule, and Authentication Behavior client services. ~212 new operations.

**Architecture:** Five new service files with per-service type files. Access Control types are shared across Door Control and Credential services. All operations follow the established callMethod pattern.

**Tech Stack:** Go stdlib, internal/soap, httptest.

**Depends on:** Phase 1 (Foundation) must be complete. Independent of Phases 2, 3, 4, 6.

**Ordering:** Access Control first (shared types), then Door Control + Schedule in parallel, then Credential, then Auth Behavior.

**Spec:** `docs/superpowers/specs/2026-03-31-onvif-full-compliance-design.md`

**WSDLs:**
- Access Control: `resources/specs/wsdl/ver10/accesscontrol/wsdl/accesscontrol.wsdl` (namespace: `http://www.onvif.org/ver10/accesscontrol/wsdl`, prefix: `tac:`)
- Door Control: `resources/specs/wsdl/ver10/doorcontrol/wsdl/doorcontrol.wsdl` (namespace: `http://www.onvif.org/ver10/doorcontrol/wsdl`, prefix: `tdc:`)
- Credential: `resources/specs/wsdl/ver10/credential/wsdl/credential.wsdl` (namespace: `http://www.onvif.org/ver10/credential/wsdl`, prefix: `tcr:`)
- Schedule: `resources/specs/wsdl/ver10/schedule/wsdl/schedule.wsdl` (namespace: `http://www.onvif.org/ver10/schedule/wsdl`, prefix: `tsc:`)
- Auth Behavior: `resources/specs/wsdl/ver10/authenticationbehavior/wsdl/authenticationbehavior.wsdl` (namespace: `http://www.onvif.org/ver10/authenticationbehavior/wsdl`, prefix: `tab:`)
